# User Flows

## Flow 1: Browse From Home To Product

1. Visitor lands on `index.html`.
2. Visitor sees Just Life brand, apparel imagery, and department buttons.
3. Visitor selects `Shop Women` or `Shop Men`.
4. Visitor arrives on the matching collection page.
5. Visitor scans product grid, prices, badges, and size options.
6. Visitor selects a size.
7. Visitor taps `Quick Add`.
8. Cart panel opens with the selected item.

## Flow 2: Category Shopping

1. Visitor opens a collection page.
2. Visitor uses category tabs such as `All`, `Tops`, `Layers`, `Outerwear`, or `Sale`.
3. Page moves to the matching product area.
4. Visitor compares visible products.
5. Visitor chooses a size and adds to cart.

## Flow 3: Cart Review

1. Visitor taps the bag button in the header.
2. Cart panel opens.
3. Visitor reviews product name, size, and price.
4. Visitor removes an item or clears the cart if needed.
5. Cart count and total update immediately.

## Flow 4: Newsletter Signup

1. Visitor scrolls to the newsletter section.
2. Visitor enters an email address.
3. Visitor taps `Subscribe`.
4. Button confirms the subscription state.

## Key UX Notes

- The cart should remain accessible from every page.
- Size selection should happen before add-to-cart to avoid incomplete cart items.
- Product badges should be used sparingly for meaningful signals: `New`, `Best Seller`, `Essential`, and `Sale`.
- Filter and sort controls are included visually, even if future behavior is added later.
